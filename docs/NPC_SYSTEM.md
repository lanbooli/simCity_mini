# NPC 系统说明

## 概述

每个 NPC 是一个独立的 Python 进程，运行"感知→意图→行动→反思"的自主循环。NPC 拥有性格、记忆、心情、关系和目标，能够自主社交、发朋友圈、与玩家互动，甚至恋爱和结婚。

## NPC 属性

| 属性 | 说明 |
|------|------|
| `name` | 名称 |
| `gender` | 性别 (male/female/other) |
| `personality` | 性格标签数组，如 `["外向","热情","浪漫"]` |
| `voice_type` | 语音类型（预留） |
| `attributes` | 体能属性 (stamina/speed/strength, 1-10) |
| `current_mood` | 当前心情 |
| `current_activity` | 当前活动描述 |
| `current_scene_id` | 所在场景 |
| `schedule` | 日程表 (JSON) |

## 性格系统

性格标签影响 NPC 的所有决策：

| 性格 | 影响 |
|------|------|
| 外向/开朗 | 社交概率 ×1.5, 主动互动 ×1.8, 发帖 ×2.5 |
| 内向/害羞 | 社交概率 ×0.5, 主动亲密 ×0.3, 发帖 ×0.3 |
| 热情 | 亲密增速 ×1.5, 社交加权 ×1.3 |
| 浪漫 | 告白概率 ×4.0, 求婚概率 ×5.0, 亲密动作 ×2.0 |
| 强势 | 被冒犯时反击概率高, 分手后不易复合 |
| 温柔/善良 | 更容易原谅, 复合率 +20% |
| 谨慎 | 求婚概率 ×0.5, 告白概率降低 |
| 话痨 | 发帖概率 ×3.0 |

## 决策循环

### 感知 (Perception)

每一决策周期 (15 游戏分钟)，NPC 通过 Redis KV 查询：
- 同场景有哪些实体 (NPC + 玩家)
- 天气、时间、场景类型
- 各实体的心情和活动状态

### 意图生成 (Brain)

基于感知数据、性格、心情和关系数值，生成意图：

| 意图 | 触发条件 |
|------|---------|
| `greet_player` | 玩家刚进入场景, 好感 >= 0 |
| `avoid_player` | 好感 <= -50, 玩家在场景中 |
| `approach_player` | 好感 >= 60, 熟悉 >= 20 |
| `socialize_npc` | 同场景有认识 NPC, 好感 > 10 |
| `inner_thought` | 约每 30 游戏分钟概率触发 |
| `move` | 日程表或探索欲望 |
| `idle` | 无特殊事件 |

### 行动执行

- **社交动作**：LLM 生成开场白 → NPC B 消费并回复 → 双向关系更新
- **玩家互动**：LLM 生成问候/动作叙事 → 发送到对话管道 → 等待玩家回应
- **回避**：NPC 移动到其他场景
- **发帖**：LLM 生成朋友圈动态 → 写入 social_post 表

### 反思 (记忆巩固)

每天结束时 (23:45)：
- importance >= 5 的短期记忆 → LLM 总结为长期记忆
- importance <= 2 的短期记忆 → 删除

## 心情系统

| 心情 | 行为修正 |
|------|---------|
| happy/excited | 社交概率 ×1.3~1.5, 告白概率 ↑ |
| neutral | 正常行为 |
| sad | 社交概率 ×0.3, 倾诉概率 ×2.0 |
| angry | 社交概率 ×0.2, 回避概率 ↑ |
| fear | 回避概率 ↑, 寻求安全场景 |

心情随时间自然衰减，受社交互动结果影响。

## 关系系统

### 三个核心数值

| 数值 | 范围 | 增长方式 | 说明 |
|------|------|---------|------|
| favorability (好感度) | -100~100 | 互动/社交 | 非线性曲线, 极端值更难提升 |
| familiarity (熟悉度) | 0~100 | 每次互动 +1 | 线性增长 |
| intimacy_comfort (亲密舒适度) | 0~100 | 同场景共处/亲密互动 | 接受亲密+增长, 拒绝-减少 |

### 递减收益

同一动作对同一 NPC 连续使用, 效果递减：
- 第 1 次: 100% 效果
- 第 2 次: 70%
- 第 3 次: 40%
- 第 4+ 次: 15%

窗口: 480 tick (8 游戏小时)

## 恋爱系统

### 心动 → 告白 → 恋人

1. 满足阈值 → 标记 `love_eligible`
2. 玩家 `/告白` 或 NPC 主动告白
3. NPC 基于性格+关系综合判定接受/拒绝
4. 接受 → `relationship_type` 变为 boyfriend/girlfriend
5. 拒绝 → 好感/舒适下降, 冷却后重试

### 恋人期

- NPC 主动互动频率从 5% 提升至 25%
- 解锁恋人专属动作: 亲吻/深吻/依偎/说情话/牵手散步/约会
- 对话风格更亲密 (`temperature=0.9`, `max_tokens=384`)
- 微互动: 路过关心、思念消息、早安晚安

### 求婚 → 结婚

1. 满足条件 → 玩家 `/求婚` 或 NPC 主动求婚
2. 接受 → `relationship_type` 变为 spouse
3. 解锁配偶专属: 同床共枕/昵称/家庭计划/早安吻

### 吃醋与排他

- NPC 必须亲眼目睹或经第三方告知才吃醋
- 分级: 轻微介意 → 明显吃醋 → 激烈反应 → 关系危机
- spouse 发现第三者 → 80% 概率直接离婚

### 分手/离婚

- `/分手` 结束恋人关系 → 进入冷静期 (480 tick)
- 离婚后果更严重: 好感 -50~80, 永久不可复合
- 温柔 NPC 更容易原谅, 强势 NPC 更难复合

## 越界行为系统

当玩家对 NPC 做出不符合关系门槛的动作时触发。

### 判定规则

```
动作要求的关系等级 - NPC 当前关系等级 >= 3 → 越界
```

### 四级严重度

| 等级 | 示例 | 好感惩罚 |
|------|------|---------|
| 轻微 | stranger 做 bestfriend 级动作 | -20 |
| 中度 | stranger 做 boyfriend 级动作 | -50 |
| 严重 | stranger 做 spouse 级动作 | -80 |
| 暴力 | 殴打/推搡任何人 | -100 |

### NPC 反应

- **害羞/内向**: 逃跑/冻结
- **温柔/善良**: 礼貌拒绝 → 严厉斥责
- **强势/高冷**: 一巴掌/反击/报警
- **力量型**: 武力反击 (属性判定)

### 连锁后果

- 目击者 gossip 传播 → 全镇声望下降
- 受害者创伤 (轻微 240 tick → 暴力永久)
- 属性反击判定 (NPC strength+speed vs 玩家)

## 朋友圈系统

NPC 每个决策周期概率发帖 (基础 3%)：
- 外向 ×2.5, 话痨 ×3.0, 害羞 ×0.3
- 内容类型: 心情分享/活动打卡/社交感慨/恋爱官宣/提问互动
- 可见性: 公开/好友/密友/仅恋人
- NPC 之间互相点赞 (60% 概率点赞熟人帖)
- 动态保留 7 游戏天
